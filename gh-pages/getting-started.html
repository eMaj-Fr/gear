Getting started

How to use and setup Gear, compatibility and stuff.

1/ Download

For a simple use of Gear, just download the precompiled version (min). You can use the minified version (gear.min.css) or the clear css.

If you plan to modify the theme (which is our goal !) we recommend to use the development version of Gear. You will find the Sass files, where you can modify the variables, parameters, all following the online documentation. You will also need a Sass compiler to save and use your changes. We recommand using Prepros or Grunt.js.

For example, you can easily change :

  • Color scheme (primary, hover,...)
  • Buttons size, borders..
  • Fonts
  • Responsive settings (grid, max sizes…)
  • Class prefix...

thus creating a unique theme, while keeping all the advantages of Gear, and whithout having to change the html/css classes.

2/ Installation

Production / precompiled version

Just drop the 'Gear' folder content in your project ; you can use the css classes right away :

<!doctype html>
	<html lang="en">
	<head>
		<meta charset="UTF-8">
		<title>My Awesome Site</title>

		<!-- VIEWPORT FOR RESPONSIVE -->
		<meta name="viewport" content="width=device-width, initial-scale=1.0">
		<!-- GEAR -->
		<link rel="stylesheet" href="dist/css/gear.css">
	</head>
	<body>
		
	</body>
</html>

Or use one of our template : -examples-
that's it !

Development version

Open the .scss files in your favorite editor and start editing parameters following the documentation. The files are located in the 'gear/scss' folder. Then you can compile your .scss into working css, and edit your html. No big deal !

4/ Things you can change !

Because it's the main feature of Gear, here is a (little) list of things you can easily customize. Basically, the Sass functions of Gear have default parameters, and/or default values. By changing these values you take control of the css output.

  • Prefix

    You can prefix your gear classes. That mean, you can use it along another framework without conflicts. Gear will automatically prefix everything without you noticing. Yeah, that's cool.

  • Buttons

    Buttons are a big part of a flat webdesign. If you don't like the default buttons, here is what you can do :

    • colors
    • border radius
    • border color
    • sizes
    • padding
    • hover
    • …
  • Css reset

    Included by default, you can choose between 3 different resets.

  • Fonts

    Include all your favorite fonts like you would in a classic css file.

  • ...And much more, swith to the doc to discover all Gear capabilities.
